# numerical-methods

Программная реализация численных методов на Python (NumPy / Matplotlib): методы оптимизации, интерполяции и аппроксимации функций, нахождения собственных чисел и векторов матриц, вычисления определённого интеграла и решения задачи Коши для систем обыкновенных дифференциальных уравнений (СОДУ).

Каждый модуль покрывает отдельный раздел численного анализа и содержит как сами алгоритмы, так и вспомогательные функции для тестирования, сравнения точности методов и визуализации результатов (графики, таблицы). Примеры использования собраны в `examples.ipynb`.

## Описание модулей

### `optimization.py` — методы оптимизации

Безусловная оптимизация функций одной и нескольких переменных.

- **`golden_section_search(func, a, b, eps=1e-5)`** — метод золотого сечения для одномерной минимизации функции на отрезке `[a, b]`.
- **`simplex_method(func, x_start=None, size=None, alpha=1., gamma=2., beta=0.5, delta=0.5, eps=1e-5, count_iters=False)`** — метод Нелдера–Мида (деформируемого многогранника) для безградиентной многомерной минимизации с операциями отражения, растяжения, сжатия и редукции симплекса.
- **`Gradient_Descent`** — класс, реализующий градиентный спуск с постоянным шагом обучения (`lr`). Численный градиент вычисляется методом центральных разностей, если не передана явная функция градиента. Поддерживает несколько критериев останова: по норме градиента, по изменению значения функции и по изменению аргумента.
- **`Steepest_Descent(Gradient_Descent)`** — наискорейший спуск: длина шага на каждой итерации подбирается одномерной минимизацией (`golden_section_search`) вдоль антиградиента.
- **`Conjugate_Gradient(Gradient_Descent)`** — метод сопряжённых градиентов (Флетчера–Ривса) с периодическим перезапуском направления спуска каждые `n` итераций.
- **`Newtone_Method(Gradient_Descent)`** — метод Ньютона с использованием матрицы вторых производных (гессиана) и одномерным поиском оптимального шага вдоль ньютоновского направления.

### `interpolation.py` — интерполяция функций

Построение интерполяционных полиномов и сплайнов, сравнение их точности.

- **`create_table_uniform(func, bounds, n_nodes)`** / **`create_table_chebyshev(func, bounds, n_nodes)`** — построение таблицы значений функции на равноотстоящих узлах или узлах Чебышёва.
- **`create_lagrange_polynomial(table)`** — интерполяционный полином Лагранжа.
- **`create_newton_polynomial(table)`** — интерполяционный полином Ньютона (с разделёнными разностями).
- **`create_s10_spline(table)`** — линейный интерполяционный сплайн `S1,0`.
- **`create_s21_spline(table, d)`** — квадратичный сплайн `S2,1` с заданным граничным условием `d`.
- **`create_s32_spline(table)`** — кубический сплайн `S3,2` (естественный кубический сплайн).
- **`solve_by_spline(spline_table, points_table)`** — вычисление значений сплайна в заданных точках.
- **`compare_lagrange_and_newton(func, nm_pairs, bounds)`** — сравнение полиномов Лагранжа и Ньютона на равноотстоящих узлах и узлах Чебышёва с выводом графиков и таблиц максимальных отклонений.
- **`compare_splines_and_newton(func, nm_pairs, bounds, d)`** — сравнение линейного, квадратичного и кубического сплайнов между собой и с полиномом Ньютона.

### `approximation.py` — аппроксимация функций (МНК)

Полиномиальная аппроксимация методом наименьших квадратов (МНК).

- **`norm_mnk(table, n)`** — построение аппроксимирующего полинома степени `n` через систему нормальных уравнений МНК.
- **`ort_mnk(table, n)`** — построение аппроксимирующего полинома степени `n` с использованием ортогональных многочленов (рекуррентное построение базиса).
- **`mnk_test(func, bounds, n_nodes, values_koef=3)`** — генерирует тестовую таблицу «экспериментальных» точек и строит графики/таблицу сравнения суммы квадратов ошибок для обоих подходов МНК при степенях полинома от 1 до 6.

### `integration.py` — численное интегрирование

Класс **`Integration`** объединяет набор формул численного интегрирования и методов оценки погрешности.

Параметры конструктора: `func` (подынтегральная функция), `find_mu` (функция вычисления моментов для весовых квадратур), `bounds` (отрезок интегрирования), `eps` (требуемая точность).

- **`midpoint_rectangles(n)`**, **`left_rectangles(n)`**, **`trapezoidal_rule(n)`**, **`simpsons_rule(n)`** — классические квадратурные формулы прямоугольников, трапеций и Симпсона.
- **`newton_cotes(z1, z2, n)`** — обобщённая квадратурная формула Ньютона–Котса с весовой функцией (моменты вычисляются через `find_mu`).
- **`gaussian(z1, z2, n)`** — квадратурная формула типа Гаусса с весовой функцией; узлы находятся как корни ортогонального многочлена через формулу Кардано.
- **`skf(method, n)`** — составная квадратурная формула (разбивает отрезок на `n` частей и применяет `newton_cotes` или `gaussian` на каждой).
- **`richardson(approximations, leading_order, h, ratio=2)`** — уточнение результата методом Ричардсона и оценка погрешности по серии приближений с разным шагом.
- **`aitken(approximations, ratio)`** — оценка реального порядка точности по процессу Эйткена.
- **`skf_error_estimation(method, n_start=1, ratio=2)`** — автоматический подбор числа разбиений для достижения точности `eps` с уточнением по Ричардсону.
- **`find_optimal_n(method, ratio=2)`** — оценка оптимального стартового числа разбиений.
- **`integration_test_unweighted(true_value)`** / **`integration_test_weighted(true_value)`** — построение графиков зависимости абсолютной погрешности от числа разбиений для всех реализованных формул и вывод оптимальных шагов разбиения в консоль.

### `eigen.py` — собственные числа и векторы

Итерационные методы вычисления собственных значений и собственных векторов матриц.

- **`power_method(matrix, rtol=1e-6, delta=1e-8)`** — степенной метод: нахождение наибольшего по модулю собственного числа и соответствующего собственного вектора.
- **`inverse_power_method(matrix, start_sigma, max_value, rtol=1e-6, delta=1e-8)`** — обратный степенной метод со сдвигом `sigma` для уточнения собственного числа, ближайшего к заданному значению.
- **`find_by_inverse_power_method(matrix, bounds, rtol=1e-6)`** — поиск всех собственных чисел и векторов матрицы перебором сдвигов из заданного диапазона `bounds` с проверкой на повторяющиеся значения.
- **`qr_algorithm(matrix, rtol=1e-6, eps=1e-8)`** — QR-алгоритм со сдвигами для нахождения всех собственных чисел: предварительное приведение матрицы к форме Хессенберга (через `_to_hessenberg`) и итеративная QR-декомпозиция (через `_qr_decomposition`) с дефляцией.
- **`generate_matrix(n, bounds)`** — генерация тестовой матрицы размера `n × n` с заданным диапазоном собственных чисел `bounds`, удобной для тестирования методов выше.

### `sode.py` — решение СОДУ (задача Коши)

Класс **`SODE_Solver`** решает задачу Коши для системы ОДУ `y' = f(x, y)` явными методами Рунге–Кутты 2, 3 и 4 порядка (`order`).

Параметры конструктора: `func` (правая часть системы), `x_start`, `x_end` (границы интегрирования), `y_start` (начальные условия), `order` (порядок метода Рунге–Кутты: 2, 3 или 4), `total_err`, `local_err` (требуемые точности).

- **`const_step(h=None, do_log=False, total_err=None)`** — интегрирование с постоянным шагом `h` (либо автоматически подобранным начальным шагом).
- **`const_step_total_error(total_err=None)`** — интегрирование с постоянным шагом и контролем суммарной погрешности методом Рунге (последовательное уменьшение шага вдвое до сходимости).
- **`auto_step(h_start=None, do_log=False, total_err=None, local_err=None)`** — интегрирование с автоматическим выбором шага по контролю локальной погрешности (правило Рунге, удвоение/деление шага).
- **`show_const_step()`** — график истинной полной погрешности решения с постоянным шагом.
- **`show_auto_step()`** — графики изменения шага и отношения локальных погрешностей при автоматическом выборе шага, а также график зависимости числа вычислений правой части от требуемой точности.
- Объект ведёт лог (`self.log`) с узлами, значениями, шагами и погрешностями интегрирования для последующего анализа и построения графиков.
